# Week 3 – Session 3  
## API Troubleshooting & Project 1 Clarifications

This session mainly focused on resolving API issues, clarifying Project 1 requirements, and discussing deployment problems.

### API Keys & AI Pipe

- API key errors were mostly due to expired or incorrectly configured keys.
- Important to store keys using environment variables.
- OpenAPI keys and AI Pipe tokens are different and should not be mixed.
- Always check HTTP responses and status codes for debugging.

### Deployment Issues (Vercel)

Common FastAPI deployment problems:

- Wrong port binding.
- Missing environment variables.
- Incorrect start command.
- Timeout issues.

Most failures were configuration-related rather than code issues.

### Project 1 Requirements

- Must include FastAPI, scraping, and Discourse API usage.
- Proper deployment and working endpoints are required.
- Evaluation is based on functionality, structure, and deployment.

### Prompt Engineering & GA Issues

- Clear, structured prompts improve results.
- Chrome Dev Tools can help debug request issues.
- Many GA errors were configuration or token-related.

### Key Takeaway

This session reinforced the importance of:
- Proper API key management
- Correct environment configuration
- Deployment debugging skills
- Clear understanding of Project 1 expectations

It emphasized practical troubleshooting and real-world setup rather than new technical concepts.
